XSS

Cross-site scripting (XSS) is a web security flaw that lets attackers compromise user interactions with a vulnerable application. It allows attackers to bypass the same origin policy, which separates different websites. XSS can enable attackers to impersonate users, perform actions on their behalf, access their data, and potentially gain full control over the applicationâ€™s functionality and data if those users have privileged access.

Username Enumeration

Username enumeration is a security vulnerability where attackers identify valid usernames on a server or web application, often facilitated by different server responses for existing and non-existing users. This process can also occur through exploitable flaws like SQL Injection that reveal usernames. Knowing usernames is crucial since it's typically half of the login credentials needed, enabling attackers to subsequently employ Brute Force to guess passwords.

SQL Injection

SQL injection (SQLi) is a web security vulnerability that allows an attacker to interfere with the queries that an application makes to its database. This can allow an attacker to view data that they are not normally able to retrieve. This might include data that belongs to other users, or any other data that the application can access. In many cases, an attacker can modify or delete this data, causing persistent changes to the application's content or behavior.
